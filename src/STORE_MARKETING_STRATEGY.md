## 🎨 ERAS AESTHETIC ANALYSIS

### Visual Language:
- **Colors:** Purple/pink gradients (brand signature), slate/dark backgrounds, gold/amber accents
- **Effects:** Glassmorphism, backdrop blur, subtle animations, particle effects
- **Typography:** Clean sans-serif, dramatic size contrasts, gradient text for premium elements
- **Animations:** Motion/Framer Motion, smooth transitions, confetti celebrations
- **Tone:** Premium but approachable, magical but functional, emotional but not saccharine

### Design Principles:
1. **Non-Intrusive** — Modals fade in gently, never jarring
2. **Contextual** — Features appear when relevant (not random spam)
3. **Celebratory** — Unlocks/achievements trigger joy, not guilt
4. **Transparent** — Clear pricing, no hidden fees, honest value
5. **Elegant** — Cinema-quality VFX expectations = premium presentation

### What Works in Eras:
- ✅ Subtle gradient overlays on modals
- ✅ Pulsing badges/icons for attention
- ✅ Bottom-positioned CTAs (mobile-friendly)
- ✅ Emoji + icon combinations
- ✅ "Just unlocked" / "New" badges
- ✅ Progress bars with gradients
- ✅ Particle confetti on success

### What Doesn't Fit:
- ❌ Aggressive popups blocking content
- ❌ Countdown timers / fake scarcity
- ❌ Flashing banners / garish colors
- ❌ "BUY NOW" shouting
- ❌ Interrupting user flow mid-task

## 🧠 SMART SUGGESTION ENGINE

### Algorithm: `getSuggestedThemes(capsule, userHistory)`

**Input:**
- Current capsule theme, title, message
- User's purchase history
- User's capsule creation patterns
- Current date (seasonal context)

**Output:**
- 3 most relevant themes to suggest
- Priority scoring (1-10)

**Logic:**
```typescript
function getSuggestedThemes(
  capsule: Capsule,
  userHistory: UserHistory,
  currentDate: Date
): ThemeSuggestion[] {
  const suggestions: ThemeSuggestion[] = [];
  
  // 1. Keyword Detection (highest priority)
  const keywords = extractKeywords(capsule.title, capsule.message);
  if (keywords.includes('wedding')) suggestions.push({ theme: 'wedding', score: 10 });
  if (keywords.includes('baby')) suggestions.push({ theme: 'new_life', score: 10 });
  // ... etc
  
  // 2. Seasonal Relevance
  if (isNearNewYear(currentDate)) suggestions.push({ theme: 'new_year', score: 8 });
  if (isSummer(currentDate)) suggestions.push({ theme: 'travel', score: 7 });
  
  // 3. User Pattern Analysis
  const mostUsedFreeTheme = userHistory.getMostUsedTheme();
  if (mostUsedFreeTheme === 'birthday') suggestions.push({ theme: 'new_life', score: 6 });
  
  // 4. Bundle Recommendations (if user has multiple related capsules)
  const capsuleTypes = userHistory.getCapsuleTypes();
  if (capsuleTypes.travel >= 3) suggestions.push({ bundle: 'celebration', score: 7 });
  
  // 5. Complementary Themes (what they don't have)
  const purchasedThemes = userHistory.getPurchasedThemes();
  const unpurchasedThemes = ALL_THEMES.filter(t => !purchasedThemes.includes(t));
  suggestions.push(...unpurchasedThemes.map(t => ({ theme: t, score: 5 })));
  
  // Sort by score, return top 3
  return suggestions.sort((a, b) => b.score - a.score).slice(0, 3);
}
```

## 📊 FREQUENCY & THROTTLING RULES

### **Golden Rule: Never Annoy the User**

**Throttling Strategy:**
1. **Max 1 promotional element per session**
   - If user dismisses Post-Ceremony card, don't show Floating Hint same session
   
2. **Max 1 promotional element per day**
   - localStorage: `eras_last_store_promo_shown: timestamp`
   
3. **Never interrupt active tasks**
   - Don't show store promo while user is typing capsule
   - Wait until task completion (capsule saved/sent)
   
4. **Respect dismissals**
   - If user clicks "Maybe Later" 3 times, disable that promo type for 7 days
   
5. **Post-purchase grace period**
   - After ANY purchase, no promos for 48 hours (celebrate, don't upsell)

**Tracking Object:**
```typescript
interface StorePromoState {
  lastShown: number; // timestamp
  dismissCount: { [promoType: string]: number };
  purchaseDate: number | null;
  sessionShown: boolean;
}
```

function trackPromoDismissed(promoType: string) {
  // Increment dismiss count
  const dismissals = JSON.parse(localStorage.getItem('eras_store_promo_dismissals') || '{}');
  dismissals[promoType] = (dismissals[promoType] || 0) + 1;

  // If dismissed 3+ times, disable for 7 days
  if (dismissals[promoType] >= 3) {
    localStorage.setItem(`eras_store_promo_disabled_${promoType}`, (Date.now() + 7 * 24 * 60 * 60 * 1000).toString());

## 💡 COPYWRITING GUIDELINES

### Tone:
- **Warm, not pushy:** "You might love..." instead of "YOU MUST BUY"
- **Celebratory:** "Celebrate your milestones in style ✨" not "Upgrade now!"
- **Helpful:** "💡 Pro Tip:" frames suggestions as friendly advice
- **Honest:** Clear pricing, no hidden fees, transparent value
- **Aspirational:** "Building a legacy" resonates with Eras' mission

### Examples:

**✅ GOOD:**
- "Loved that ceremony? See more cinema-quality VFX"
- "You're building a legacy network! 🌟"
- "Celebrate your 10th capsule in style"
- "Planning a wedding capsule? Try Golden Moments for $2.99"

**❌ BAD:**
- "UNLOCK NOW OR MISS OUT!"
- "Limited time offer! Buy before it's gone!"
- "You need this to unlock your capsule's full potential"
- "Don't be basic — upgrade today!"

## 🎯 FINAL RECOMMENDATIONS

### **Top 3 Must-Haves:**

1. **Beneficiary Upgrade Modal** 🔥🔥🔥🔥🔥
   - Blocking use case = instant need
   - Highest conversion potential (50-70%)
   - Simple to build, huge impact
   - **Build this FIRST**

2. **Post-Ceremony Store Teaser** 🔥🔥🔥🔥
   - Users just saw VFX quality (warm lead)
   - Contextual suggestions feel helpful, not pushy
   - Good conversion potential (10-15%)
   - Fits Eras aesthetic perfectly

3. **Smart Theme Suggestions (Keywords)** 🔥🔥🔥
   - Feels like AI magic (even though it's simple regex)
   - Non-blocking, user can ignore
   - Adds value to capsule creation experience
   - Medium conversion (8-12%), but great UX

### **Skip These (Too Pushy for Eras):**
- ❌ Floating Store Hint (bottom-right bubble) — Too intrusive
- ❌ Countdown timers / "Limited time" messaging — False scarcity feels cheap
- ❌ Automatic popups on idle — Interrupts user flow
- ❌ Notifications/emails pushing store — Spammy

### **Philosophy:**
**"Promote when relevant, celebrate when earned, suggest when helpful."**

Eras is a premium, emotional product. Users are preserving life memories, not shopping for widgets. Store promotions should feel like:
- A helpful suggestion from a friend
- A natural extension of the celebration (achievements)
- An enhancement opportunity, not a requirement

**Never make users feel like they're missing out.** The free themes are beautiful. Premium themes are *even better*, but not essential. Frame it as "unlock more magic" not "what you have isn't enough."

## 📞 QUESTIONS TO CONSIDER

1. **Conversion Target:** What % of users do you hope will purchase?
   - Industry standard for freemium: 2-5%
   - Premium apps: 8-15%
   
2. **Revenue Goal:** What monthly revenue target?
   - Helps prioritize high-value promos (bundles vs. impulse)
   
3. **User Tolerance:** How many promos is too many?
   - Start conservative (1 per week max), increase if well-received
   
4. **Discount Strategy:** Ever offer sales/promotions?
   - "Black Friday bundle discount" could be huge
   - But avoid constant sales (devalues product)

5. **Referral Integration:** Should referrals unlock store credits?
   - "Refer 3 friends → $2 store credit"
   - Could boost both referrals AND purchases
